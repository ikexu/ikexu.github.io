Java基础之JDBC

发表于 2016-09-21   |   分类于 java   |  

概念

MyJDBC（Java Data Base Connectivity,java数据库连接）是一种用于执行SQL语句的Java API，可以为多种关系数据库提供统一访问，它由一组用Java语言编写的类和接口组成。JDBC提供了一种基准，据此可以构建更高级的工具和接口，使数据库开发人员能够编写数据库应用程序。
它提供了访问各种数据库（我们使用Mysql）的通用接口，帮助程序猿与数据库建立连接、发送操作数据库的语句并处理结果。

JDBC主要有以下几个步骤：

1.加载及注册JDBC驱动程序
2.JDBC URL 定义驱动程序与数据源之间的连接
3.建立连接对象
4.建立SQL陈述式对象（Statement Object）
5.执行SQL语句

加载及注册JDBC驱动程序
Class.forName("com.mysql.jdbc.Driver");

JDBC URL 定义驱动程序与数据源之间的连接
String url = "jdbc:mysql://localhost:3306/jdbctest?user=root&password=123&useUnicode=true&characterEncoding=UTF8";
​
这里的user及password是你数据库的用户名和密码，除此之外还有其余可选参数
标准语法：jdbc:mysql//[hostname][:port]/[dbname][?param1=value1][&param2=value2]….

建立连接对象
Connection con = DriverManager.getConnection(url);

建立SQL陈述式对象（Statement Object）
Statement stmt = con.createStatement();

进阶：连接池优化

上面所说的数据库连接方式，也有一些弊病，用户每次操作请求都要重新获得连接，而数据库建立连接又是非常耗费资源的，数据量比较小时不明显，一旦数据量非常大时系统就会非常迟钝，性能低下，甚至数据库服务器内存溢出、拓机等，因此我们使用连接池来优化。

使用连接池优化后会是这样

可以看出多了一个负责分配管理connection的连接池，connection的获得由直接向数据库请求变成了向连接池请求，由连接池与数据库交互，连接池中的connection是可以重复使用的，用户请求后会分配一个连接，使用结束后还给连接池，而不是每次都新建一个connection。
数据库连接池在初始化时将创建一定数量的数据库连接放到连接池中, 这些数据库连接的数量是由最小数据库连接数来设定的.无论这些数据库连接是否被使用,连接池都将一直保证至少拥有这么多的连接数量.连接池的最大数据库连接数量限定了这个连接池能占有的最大连接数,当应用程序向连接池请求的连接数超过最大连接数量时,这些请求将被加入到等待队列中.
实现连接池我们可以自己实现，需实现java.sql.DataSource接口，大家了解原理就行，不必自己实现。我们采用别人实现好的开源连接池如DBCP、C3P0连接池等，我们将介绍DBCP连接池。
DBCP 是 Apache 软件基金组织下的开源连接池实现，要使用DBCP数据源要增加两个jar包：

Commons-dbcp.jar：连接池的实现
Commons-pool.jar：连接池实现的依赖库
